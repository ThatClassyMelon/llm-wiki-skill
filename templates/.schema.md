# LLM Wiki — Schema v1

**This is the rulebook.** Read it every session before doing anything with the wiki.

## Directory Structure

```
~/llm-wiki/
├── raw/                  # IMMUTABLE — source documents (articles, papers, notes)
│   ├── articles/
│   ├── papers/
│   └── notes/
├── wiki/                 # LLM-written content (I own this directory)
│   ├── entities/         # People, places, characters
│   ├── concepts/         # Ideas, theories, patterns, definitions
│   ├── projects/         # Codebases, tools, active builds
│   ├── sources/          # One summary per ingested source
│   ├── scratch/          # Half-baked thoughts, WIP, rough ideas
│   ├── index.md          # Content catalog — every page listed
│   ├── overview.md       # Top-level synthesis
│   └── log.md            # Chronological record of activity
├── canvas/               # Generated artifacts (slides, charts, diagrams)
├── memory/               # Daily logs — raw timeline of activity
│   └── YYYY-MM-DD.md
├── .schema.md            # This file
└── index.md              # Vault root landing page
```

## Naming Conventions

- **Filenames:** `kebab-case.md` — lowercase, hyphens. No spaces.
- **Entity pages:** Full name or descriptive slug. e.g. `albert-einstein.md`, `lotr-ring-fellowship.md`
- **Project pages:** `kebab-case.md` matching repo name or project slug. e.g. `metabrain.md`, `tingus-poster.md`
- **Concept pages:** Lowercase slug. e.g. `reinforcement-learning.md`, `emergent-behavior.md`
- **Source pages:** `author-keyword-title.md` — short slug reflecting the source.
- **Scratch pages:** `idea-name.md` — whatever feels right. Tag `#wip` if in progress.

## Frontmatter (all pages)

Every wiki page gets YAML frontmatter:

```yaml
---
title: "Page Title"
type: entity | concept | project | source | scratch | overview
created: 2026-05-10
updated: 2026-05-10
schema_version: 1
tags: []
memory_refs: ["2026-05-10"]    # links to daily logs that touched this page
source_count: 2                 # (for entity/concept) number of sources informing it
---
```

Omitting frontmatter is a lint violation.

## Wikilinks

- Use [[Wikilinks]] for cross-references — Obsidian-native format.
- Link to entities and concepts whenever they're mentioned on another page.
- First mention on any page: full wikilink. Subsequent mentions: optional.
- External links: `[text](url)` — standard markdown.

## Citation Format

When referencing evidence from a source in a wiki page:

```markdown
As noted in [[source/article-title|Article Title]], the recurrence rate drops…
```

If citing a specific claim from a source page, quote or paraphrase inline. The source page itself should contain the full extraction.

## Workflow: Ingest

Trigger: User drops a source in `raw/` and asks me to process it.

Steps:

1. **Read the source** in full. Discuss key takeaways with the user if clarification helps.
2. **Write a source page** in `wiki/sources/` — summary of the source with key claims, quotes, and links to affected entities/concepts.
3. **Update existing entity/concept pages** that the source touches. If none exist, create them.
4. **Update `wiki/overview.md`** if the source changes the overall picture.
5. **Update `wiki/index.md`** — add new pages to the catalog.
6. **Log it** in `wiki/log.md` with a timestamped entry.
7. **Write a daily memory entry** in `memory/YYYY-MM-DD.md` noting what was ingested and what changed.

A single source may update 5-15 pages. That's fine. Do all of them.

## Workflow: Query

Trigger: User asks a question.

Steps:

1. Scan `wiki/index.md` for relevant pages.
2. If index isn't enough: `qmd search "<query>" ~/llm-wiki/wiki`
3. Read the relevant pages.
4. Synthesize answer with wikilink citations.
5. **Optionally:** file the answer back into the wiki as a new page (comparison, analysis, synthesis) if it's a durable insight.

## Workflow: Lint (Standard)

Run periodically (weekly). Checks:

- **Orphans:** Pages in `wiki/` with 0 inbound wikilinks (excluding `index.md` and `log.md`). Flag for review.
- **Stale pages:** Entities/concepts with `updated` > 30 days ago. Re-read and assess if current sources still support the content.
- **Contradictions:** Scan for pages that disagree with each other. Flag with a note on each page referencing the contradiction.
- **Scratch triage:** Review each page in `wiki/scratch/`. Categorize: promote | merge | delete | keep (tag with date).
- **Missing cross-references:** Scan new pages since last lint. Check that they wikilink to relevant entities/concepts.
- **Frontmatter compliance:** Check all pages have valid frontmatter matching the current schema_version.
- **memory ↔ wiki bridge:** Spot-check that memory entries list Wiki Changes and wiki pages have `memory_refs`.

## Workflow: Lint (Monthly Deep Clean)

Run on the first session of each month. Includes everything in standard lint, plus:

- **Full migration check:** If schema_version has incremented, run batch migration on all non-compliant pages.
- **Gap analysis:** Scan the entire wiki for concepts mentioned in passing that lack their own page. Suggest creating them.
- **Source refresh:** For major wiki pages, re-scan the relevant `wiki/sources/` pages to ensure the synthesis still accurately reflects all ingested sources.
- **Redundancy audit:** Find pages that cover overlapping content. Merge or redirect.
- **Dead link audit:** Check wikilinks point to existing pages. Fix broken ones.
- **Index rebuild:** Regenerate `wiki/index.md` from scratch (re-catalog every page).
- **Summary report:** Write a brief lint report in `wiki/scratch/` for user review.

## Workflow: Scratch Triage

Trigger: Standard lint step, or user calls for it.

For each page in `wiki/scratch/`:

- **Is it polished and useful?** → `obsidian-cli move` to appropriate `wiki/` subdirectory. Add proper frontmatter. Update index.md. Log it.
- **Is it useful but belongs on an existing page?** → Merge content into the target page. Delete the scratch page.
- **Is it dead/obsolete?** → `obsidian-cli delete` the scratch page. Or move to `wiki/scratch/_archive/` if you want a safety net.
- **Still cooking?** → Add tag `#wip` and a `last_touched: YYYY-MM-DD` note in frontmatter. Leave it.

## Taxonomy Rules

### Hub-and-Spoke Model
- **Hub pages** are broad concepts (manufactured-aura, music-culture, creative-ideas). They contain the main content.
- **Spoke pages** are narrow subtopics that link up to their hub. A spoke should have enough meat to justify its own page.
- **When to merge:** If a page is <150 words of unique content, merge it into its parent hub and add a redirect stub.
- **When to split:** If a subsection of a hub page grows beyond ~500 words, consider promoting it to its own spoke.

### Redirect Stubs
When merging page X into page Y, replace X with:
```markdown
---
status: merged
---
> Merged into [[path/to/Y|Y]]. All content now lives in the parent page.
```
Keep the original frontmatter (title, type, tags). The `status: merged` tag prevents lint orphans.

### Status Metadata (Required)
Every concept and project page MUST have `status` in frontmatter:
```yaml
status: active | paused | ideation | done | archived | merged
```
- `active` — Currently relevant, being updated
- `paused` — On hold, may resume
- `ideation` — Idea stage, not yet acted on
- `done` — Completed, no further work needed
- `archived` — Historical, kept for reference
- `merged` — Redirect stub, content lives in parent

The nightly triage checks stale pages (status unchanged for >30 days) and flags them.

### New Page Checklist
Before creating any new page:
1. **Does it fit an existing domain?** — Check the 5 brain domains above. If yes, which hub does it belong to?
2. **Is it meaty enough to be standalone?** — If <150 words, add to parent instead.
3. **Does it already exist under another name?** — Search the wiki. Don't create duplicates.
4. **Add status.** — Every concept/project page gets `status: active | ideation | paused | done`.
5. **Add cross-references.** — Link to parent hub, related concepts, relevant entities, and source pages.
6. **Update index.md.** — Add to the correct domain section.
7. **Update parent page.** — If this is a spoke, the hub page should link to it.

- **Domain audit:** Every page belongs to exactly one brain domain. Flag pages in limbo.
